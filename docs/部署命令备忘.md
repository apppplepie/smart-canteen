# SCS 部署命令备忘

## 一、在你电脑上（只做一次：改代码 → 打包 → 推送）

在**仓库根目录**执行，**不用**再写 `BACKEND_UPSTREAM`，后端地址写在**前端服务器**的 `.env` 里即可。

```powershell
cd d:\workspace\code\test\scs
docker compose -f docker-compose.frontend.yml build
docker push creepender42/scs-frontends:latest
```

（用户名若不是 `creepender42`，把镜像名改成你的。）

---

## 二、前端服务器 `198.23.254.84`（放 compose + `.env`）

目录例如 `/srv/scs/`，文件：`docker-compose.yml`（可用 `docker-compose.frontend.deploy.yml` 内容）+ `.env`。

**`.env` 示例：**

```env
DOCKERHUB_USER=creepender42
# 必填：Spring Boot 根地址（不要尾斜杠）
BACKEND_UPSTREAM=http://74.48.91.105:8081
# 可选：页面端口，默认 9080
FRONTEND_HOST_PORT=9080
```

**命令：**

```bash
cd /srv/scs
docker compose pull
docker compose up -d
```

改 `.env` 里 `BACKEND_UPSTREAM` 后执行：

```bash
docker compose up -d --force-recreate
```

**浏览器 502、但容器里 `wget http://74.x:8081/api/...` 正常：**  
说明 Nginx 仍在用错的地址（常见：没 `--force-recreate`，或 `.env` 写了引号）。先执行：

```bash
docker exec scs-frontends cat /etc/nginx/conf.d/default.conf | grep proxy_pass
```

应看到 `proxy_pass http://74.48.91.105:8081;`。若是 `172.17.0.1`，检查 `.env` 无引号、一行 `BACKEND_UPSTREAM=http://74.48.91.105:8081`，再 `docker compose up -d --force-recreate`。

## 三、后端服务器 `74.48.91.105`

这里**不写** `BACKEND_UPSTREAM`（那是给前端容器用的）。

按你原来的方式起 Spring Boot / `docker-compose.deploy.yml` 即可；保证 **8081**（或你实际端口）对 **198 那台公网 IP** 或全网可达（防火墙放行）。

后端自己的数据库、密钥等仍在 **`backend` / `ai-service` 各自目录或 compose 的 `.env`**，与前端 `.env` 无关。

---

## 四、两台机分工小结

| 机器        | `.env` 里主要写什么 |
|------------|---------------------|
| **198 前端** | `BACKEND_UPSTREAM` → 指向 74 的后端；`DOCKERHUB_USER`；可选 `FRONTEND_HOST_PORT` |
| **74 后端**  | 数据库、JWT、`DEEPSEEK_API_KEY` 等（原有配置，不变） |
